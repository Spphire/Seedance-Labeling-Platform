# Seedance 标注员 SOP

适用对象：负责提交 episode、检查生成结果、保留或重跑 clip 的标注人员。

标注入口：`http://106.14.2.243:18080/label`

管理员入口：`http://106.14.2.243:18080/admin`

## 1. 开始前

1. 使用 Chrome 或 Edge 打开平台。
2. 在右上角“审核人”输入自己的名字，方便别人看到锁是谁持有。
3. 标注页可以微调“本次 Prompt”和“本次参考图顺序”，这些只影响本次生成或重跑。
4. 管理员入口用于调整全局默认 prompt、ref 图片顺序、API Key 和查看监控；标注页每次打开会从管理员默认值重新填充。
5. 每次打开页面时，本次运行模式默认是 `mock`。只有负责人明确要求时，才切到 `seedance`，真实模式会消耗额度。

## 2. 锁定规则

- 打开或操作一个 clip 时，页面会申请 clip 锁。
- 持有锁的人才能保留、丢弃、重跑、标记问题。
- 其他人看到锁定中的 clip 时只能查看，不能提交审核操作。
- 同一个浏览器 profile 的多个标签页通常算同一个标注员；不同浏览器、无痕窗口、不同浏览器 profile 会被当成不同标注员。
- 锁会自动续期。关闭页面时会尝试释放；异常关闭后锁会在一段时间后自动过期。

## 3. 提交和预处理 episode

推荐在左侧“批量 episode”文本框中一行填写一个 UUID：

```text
5fe70851-34d3-424f-a2a2-6721a24f25fd
869a1118-5e18-4ebe-82fe-778d515c2688
```

也可以用空格或逗号分隔；系统会从文本中提取标准 UUID，并自动去重。

操作步骤：

1. 把 episode UUID 粘贴到文本框。
2. 点击“提交并预处理”。
3. 平台会拉取数据、提取 head 视频、切分 clips。
4. 如果某个 episode 已经完整切好，系统会跳过它，继续处理后面的 episode。

如果文本框为空但当前选中了一个 episode，点击“提交并预处理”会重新处理当前选中的 episode。这个主要用于修复损坏 episode。

## 4. Episode 标签含义

- `pending`：已提交，尚未完成预处理。
- `preprocessing`：正在预处理。
- `preprocessed`：head 视频和 clips 已准备好。
- `damaged` / `损坏`：已预处理记录存在，但 head 文件、clip 文件或切片文件缺失，建议选中该 episode 后重新预处理。
- `failed`：预处理失败，查看底部日志或联系负责人。
- `ready`：全部保留后，final episode 已合成。
- `stitching`：全部保留后正在合成 final。

注意：`final_status=missing` 只表示 final 视频还没有合成，不代表预处理失败。

## 5. 批量生成

左侧“生成”会批量处理需要生成的 clips：

- 会处理：`pending`、`generated_failed`、`rejected`。
- 不会处理：`generated`、`accepted`、`flagged`、`generating`。
- 被别人锁定的 clip 会被跳过，避免覆盖正在审核的工作。

`mock` 模式不会消耗 Seedance 额度，会模拟真实耗时并复制原 clip 作为生成结果，适合流程检查。

`seedance` 模式会真实调用 Seedance API，会消耗额度。切换前必须确认本次 Prompt、ref 图片顺序和运行模式。

## 6. 审核 clip

1. 在左侧 clip 列表中选择一个 clip。
2. 等右侧加载原 clip 和生成 clip。
3. 同步播放两边视频，检查动作、背景、爪夹形态和机械臂替换效果。
4. 根据结果选择操作：

- “保留”：结果合格，进入 accepted。
- “丢弃”：当前生成结果不合格，后续批量生成会重新处理 rejected clip。
- “重跑”：立即重跑当前 clip，使用当前页面选择的运行模式、本次 Prompt 和本次参考图顺序。
- “标记问题”：数据或结果需要人工判断，批量生成不会自动重跑 flagged clip。

所有 clip 都保留后，系统会自动合成 final episode。合成完成后，episode 会显示 final 链接。

## 7. 常见问题

### 提交多个 UUID 后没有处理新 episode

刷新页面后确认文本框里有 UUID。文本框有内容时，平台会优先按文本框处理；文本框为空时才会处理当前选中的 episode。

### Episode 已经切好了，但左侧曾经显示 missing

`missing` 指的是 final 视频不存在，不是预处理缺失。现在主标签会显示 `preprocessed`；如果文件缺失，会额外显示“损坏”。

### 正在生成时为什么不能重跑

同一个 clip 正在 `generating` 时会被锁定，防止重复消耗和覆盖结果。等待当前任务完成或失败后再重跑。

### 什么时候用“丢弃”和“标记问题”

- 用“丢弃”：这个生成结果不合格，但 clip 本身可以继续重跑。
- 用“标记问题”：clip 数据、动作、画面或判断标准不清楚，需要负责人确认。

### 页面显示别人锁定

等待对方完成或锁过期。不要随意接管别人正在审核的 clip，除非负责人明确要求。

## 8. 下班或交接前

1. 完成当前 clip 的审核动作。
2. 不要让页面长时间停在持锁 clip 上。
3. 关闭页面释放锁。
4. 如果有 failed、damaged、flagged 的 episode/clip，记录 UUID 并反馈给负责人。
